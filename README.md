# Mediterranean Atomic Tales: The Sunken Land  
## Documentation Repository

Questo repository contiene la **documentazione ufficiale** del progetto  
**Mediterranean Atomic Tales: The Sunken Land (MAT:TSL)**.

MAT:TSL è un progetto open source di **worldbuilding, game design e documentazione tecnica** ambientato in un'Europa post-apocalittica (con focus sull'Italia), all'interno dell'universo narrativo di una nota Saga videoludica.

---

## Cos’è MAT:TSL
**MAT:TSL non è un videogioco pronto**, ma una base solida e strutturata per:
- sviluppare un videogioco RPG 3D open-world in prima persona
- creare un’ambientazione coerente e stabile
- supportare narrazione, quest design e level design
- permettere collaborazioni open source sul worldbuilding

Il progetto segue il principio:
> **prima il mondo, poi il gioco**

---

## Obiettivi del repository
Questo repository serve a:
- documentare **lore, storia, fazioni e mondo**
- definire **meccaniche di gioco e sistemi**
- fornire **linee guida tecniche e produttive**
- mantenere **coerenza e tracciabilità delle decisioni**

Quando la documentazione raggiungerà la versione **1.0.0**, l’ambientazione sarà
considerata **canonica e stabile**.

## Struttura della documentazione

```text
docs/
├── designdocs        # Game design, sistemi, meccaniche e aspetti tecnici
├── gamebible         # Lore, worldbuilding e coerenza narrativa
├── playerguide       # Documentazione per il giocatore finale
├── production        # Gestione progetto, QA, roadmap e tracking
├── references        # Fonti, ispirazioni e materiale di supporto
├── templates         # Template e scheletri per nuovi documenti
└── complete_index.md # Indice generale navigabile
```

### Note sulla struttura
- __designdocs/__ Documentazione per designer, programmatori e artisti (meccaniche, AI, economia, UI/UX, asset, rendering, audio).
- __gamebible/__ Cuore del progetto: ogni luogo, evento, fazione o personaggio deve essere definito qui prima di essere usato nel gameplay.
- __playerguide/__ Documentazione in-universe o rivolta al giocatore finale, scritta in linguaggio semplice.
- __production/__ Gestione del progetto: pianificazione, QA, changelog, build e task tracking.
- __references/__ Libri, giochi, articoli, media e fonti di ispirazione.
- __templates/__ Template markdown per mantenere coerenza nella documentazione.

## Filosofia del progetto
- Coerenza narrativa prima dell’estetica
- Cause ed effetti sempre espliciti
- Tecnologia come errore sistemico, non come salvezza
- Post-apocalisse consapevole, non tribale
- L’Italia come protagonista, non come sfondo esotico

Per una visione completa del progetto, vedi il file [VISION.md](VISION.md)
