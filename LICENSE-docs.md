#  Licenza: Documentazione vs Codice (fondamentale)
Qui facciamo una distinzione __molto importante__ fra la documentazione e il codice.

---

## Documentazione & Worldbuilding
### Licenza consigliata
__Creative Commons Attribution-NonCommercial-ShareAlike 4.0__
```
**CC BY-NC-SA 4.0**
```

### Cosa permette
- chiunque può leggere e condividere  
- può adattare e modificare  
- __non può usarla commercialmente__  
- le opere derivate devono usare la __stessa licenza__

### Perché è perfetta per la documentazione di MAT:TSL
- Protegge il worldbuilding
- Evita appropriazioni commerciali
- Favorisce collaborazione
- È standard nei progetti creativi

File LICENSE nel repository [MAT-TSL-documentation](https://github.com/villo001/MAT-TSL-documentation)

## Codice (quando arriverà)
Quando si inizierà a scrivere codice vero e proprio ci sono varie opzioni, la scelta verrà fatta in quel momento e questo file verrà corretto:

### Opzione A – permissiva
**MIT License**
- Massima libertà
- Zero frizioni
- Standard assoluto

### Opzione B – più protettiva
**GPLv3**
- Impone open source anche ai derivati
- Più ideologica

---

## Regola d’oro
- **Non mischiare licenze**
- Specificare sempre:
  - documentazione -> CC
  - codice -> MIT/GPL
